# 006_2023년 8월 1주차; 나는 Qlik Sense로 데이터 로드와 BI 학습을 시작했다

2023년 8월 첫째 주의 [[ChatGPT 대화이력]]은 [[Qlik Sense]] 학습으로 다시 방향을 튼다. 앞선 몇 주가 [[수소경제]]와 [[그린스틸]] 같은 산업 전환의 언어를 다뤘다면, 이번 주는 데이터 분석 도구를 실제로 다루는 방법을 묻는다. 나는 “무엇을 알아야 하는가”에서 “도구 안에서 어떻게 데이터를 불러오고 해석하는가”로 이동했다.

시작 질문은 단순했다.

> “Qliksense에서 데이터 로드를 하는데 도움을 받고 싶어.”

이 문장은 도구 학습의 출발점이다. Qlik Sense를 열고, 앱을 만들고, 데이터 원본을 선택하고, 데이터를 로드하고, 변환하고, 다시 로드한 뒤 시각화로 넘어가는 기본 흐름을 확인했다. 여기서 중요한 것은 내가 처음부터 시각화 결과를 묻지 않았다는 점이다. 먼저 물은 것은 데이터 로드였다. 분석은 화면의 차트에서 시작되지 않고, 데이터가 어떤 구조로 들어오는지에서 시작된다는 감각이 있었다.

곧바로 나는 답변 언어를 바로잡았다.

> “앞으로 모든 답변은 한글로 해줘.”

이 요청은 단순한 번역 선호가 아니다. 실제 업무 학습에서는 언어가 이해 속도를 결정한다. Qlik Sense처럼 메뉴와 개념이 섞이는 도구를 배울 때, 설명 언어를 한국어로 고정하는 것은 반복 가능한 학습 환경을 만드는 일이다.

다음 질문은 더 구체적이었다.

> “Qliksense에서 데이터 로드 편집기를 활용해서 불러온 rawdata를 조회하는 방법을 알려줘.”

여기서 관심은 [[데이터 로드 편집기]]로 좁혀진다. 데이터를 불러온 뒤 테이블과 필드를 확인하고, 데이터 미리보기로 원자료 일부를 검토하며, 필요한 경우 필터링과 편집을 적용하는 절차를 확인했다. 나는 Qlik Sense를 결과 대시보드 도구로만 보지 않고, 로드된 데이터의 구조를 확인하는 분석 준비 공간으로 이해하려 했다.

이번 주의 중심 장면은 스크립트 해석이다. 나는 실제 Qlik Sense 데이터 로드 스크립트를 붙여 넣고 이렇게 물었다.

> “Qliksense의 데이터 로드 편집기에 입력되 ㄴ스크립트야. 이걸 해석해줘.”

스크립트에는 `QuartersMap`, `Temp`, `TempCalendar`, `MasterCalendar`가 등장했다. `QuartersMap`은 월을 분기로 매핑하고, `Temp`는 `사원속성` 테이블의 `조회일자` 최소값과 최대값을 가져오며, `TempCalendar`는 날짜 범위를 생성하고, `MasterCalendar`는 조회일자·년도·월·일·년월·분기·주차·요일 같은 시간 차원을 만든다. 나는 이 과정을 통해 [[마스터 캘린더]]가 단순한 날짜 목록이 아니라 분석 가능한 시간 축을 만드는 데이터 모델링 장치라는 점을 확인했다.

곧이어 나는 기간을 업무 목적에 맞게 바꾸는 방법을 물었다.

> “Temp 테이블의 기간을 내가 원하는대로 설정하도록 하려면 어떻게 수정해야 하니? 예를들어 2010년 1월 1일부터 현재 날짜까지로 바꾼다면?”

이 질문은 스크립트를 읽는 단계에서 고치는 단계로 넘어간다. 데이터의 최소·최대 날짜를 원천 테이블에서 자동으로 가져오는 대신, `MakeDate(2010, 1, 1)`과 `Today()`를 활용해 분석 기간을 명시적으로 지정하려는 흐름이다. 분석 도구를 배우는 과정이 메뉴 안내에서 스크립트 제어로 이동한 것이다.

같은 주에는 Qlik Sense 핸드북도 등장한다. 나는 `Hands-On Business Intelligence with Qlik Sense` 자료를 가져와 요약을 요청했고, 이후 다시 한국어 정리를 요구했다.

> “내용을 한글로 정리해줘.”

정리된 핵심은 [[셀프서비스 분석]], 데이터 로딩과 모델링, 분석 애플리케이션 구축, 추가 기능이었다. 이 흐름은 내가 Qlik Sense를 단발성 문제 해결 도구가 아니라 학습해야 할 BI 체계로 보려 했다는 흔적이다. 단순히 지금 막힌 스크립트만 해결하려는 것이 아니라, 도구의 전체 구조를 이해하려는 방향이었다.

이번 주의 대화는 [[HR Analytics]]와도 연결된다. 원문 스크립트에는 `사원속성`, `조회일자` 같은 필드가 보인다. 즉 나는 HR 데이터를 다루기 위한 분석 환경을 만들고 있었을 가능성이 높다. Qlik Sense의 데이터 로드, 마스터 캘린더, 기간 설정은 모두 사람 데이터의 시계열 분석을 가능하게 만드는 기반 작업이다.

이 주차의 의미는 도구 숙련의 시작이다. 나는 ChatGPT에게 일반 개념 설명만 요구하지 않았다. 실제 스크립트를 붙여 넣고, 그 코드가 무엇을 하는지 해석시키고, 원하는 기간으로 고치는 방법까지 물었다. [[학습루프]]는 “도구 소개 → 한국어 환경 설정 → rawdata 조회 → 스크립트 해석 → 기간 수정 → 참고자료 요약”으로 이어졌다.

이 흐름은 이후의 데이터 분석 업무와 연결되는 기초 체력이다. 좋은 분석은 좋은 시각화보다 앞서 좋은 데이터 로드와 시간 축 설계에서 시작된다. 2023년 8월 첫째 주의 나는 그 출발점을 Qlik Sense 안에서 확인하고 있었다.

## 그래프뷰 관계

- [[ChatGPT 대화이력]] → [[Qlik Sense]]: 이번 주 대화는 ChatGPT를 Qlik Sense 학습 보조자로 활용한 기록이다.
- [[Qlik Sense]] → [[데이터 로드 편집기]]: 데이터 분석의 시작점은 원자료를 불러오고 구조를 확인하는 로드 편집기였다.
- [[데이터 로드 편집기]] → [[마스터 캘린더]]: 실제 스크립트 해석을 통해 시간 차원을 만드는 구조를 이해했다.
- [[마스터 캘린더]] → [[데이터 모델링]]: 날짜, 주, 월, 분기, 주차 필드는 분석 가능한 시간 모델을 만든다.
- [[BI]] → [[셀프서비스 분석]]: Qlik Sense 핸드북 요약은 사용자가 직접 분석 앱을 만들고 해석하는 흐름을 강조했다.
- [[HR Analytics]] → [[Qlik Sense]]: `사원속성`과 `조회일자`는 HR 데이터를 BI 도구로 분석하려는 맥락을 보여준다.
- [[학습루프]] → [[데이터분석]]: 질문은 도구 소개에서 스크립트 수정까지 단계적으로 깊어졌다.
